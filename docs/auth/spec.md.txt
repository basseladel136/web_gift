# Authentication Feature

## Overview
This feature handles user authentication including registration, login, logout, and profile management.

---

## Goals
- Allow users to create an account
- Allow users to login securely
- Allow users to logout
- Allow users to view and update their profile

---

## Functional Requirements

### 1. Register
- User can register using:
  - name
  - email
  - password
- Password must be hashed before storage
- Email must be unique
- Return success message + user data (without password)

---

### 2. Login
- User logs in using email + password
- Validate credentials
- If valid → return authentication token/session
- If invalid → return error message

---

### 3. Logout
- Invalidate user session/token
- Prevent further authenticated requests

---

### 4. Profile
- User can view their profile
- User can update:
  - name
  - email
  - password (optional)
- Only authenticated users can access this endpoint

---

## API Endpoints

### Auth Routes
- POST /auth/register
- POST /auth/login
- POST /auth/logout

### User Routes
- GET /user/profile
- PUT /user/profile

---

## Validation Rules

### Register
- name: required, string
- email: required, email, unique
- password: required, min 6 characters

### Login
- email: required
- password: required

---

## Security Requirements
- Passwords must be hashed (never stored in plain text)
- Use authentication middleware for protected routes
- Prevent unauthorized access to profile endpoints

---

## Edge Cases
- Register with existing email
- Login with wrong password
- Access profile without authentication
- Logout without active session
